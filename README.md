# The ICCSSE Prompt Framework

A structured approach to writing AI prompts that get usable output on the first try.

## The Problem

90% of AI prompts fail for the same 5 reasons: no identity, no constraints, no examples, too vague, no structure.

## The Framework

| Letter | Component | What it does | Example |
|--------|-----------|-------------|---------|
| **I** | Identity | Tell the AI who to be | "You are a senior tax attorney with 20 years at a Big Four firm" |
| **C** | Context | Give background info | "I'm preparing a quarterly report for our Series B board meeting" |
| **C** | Constraints | Set limits and rules | "Under 300 words. No preamble. No filler." |
| **S** | Steps | Break into ordered steps | "First research X, then compare Y, then recommend Z" |
| **S** | Specifics | Be precise about details | "Focus on revenue impact for B2B SaaS companies" |
| **E** | Examples | Show what good looks like | "Write like this: [example]. Not like this: [example]." |

## When to Use What

- **Simple tasks** (quick questions, short edits): Use 2-3 components
- **Complex tasks** (reports, analysis, code): Use all 6

## Results After 10,000+ Prompts

- Revision rounds: 3-4 → 1-2
- First-draft usability: ~30% → ~85%
- Token cost reduction: ~60%
